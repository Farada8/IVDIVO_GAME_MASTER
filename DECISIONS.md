## Правила

- Только основатель может присвоить статус APPROVED.
- Новая гипотеза не заменяет утверждённое решение без записи в этом файле.
- Каждое фундаментальное изменение получает ID формата `DEC-000`.
- Если решение меняет главный контекст, обновляется `PROJECT_CORE_CONTEXT.md`.

### DEC-009 — Порядок построения вселенной: архитектура → философия → базовые практики → отбор → активация

Статус: PROPOSED  
Дата фиксации: 2026-07-11  
Источник: текущий диалог

Работа над вселенной Плерос/LIMINAL проходит через обязательную последовательность этапов, не пропускаемых и не меняемых местами:

1. **Архитектура вселенной** — структура уровней и их иерархических отношений (см. DEC-003, WORLD_ARCHITECTURE_v0.1_DRAFT).
2. **Философия** — внутренняя логика и принцип действия каждого уровня и элемента вселенной.
3. **Базовые практики** — то, как сущности мира взаимодействуют с архитектурой внутри лора (ритуалы, Лучи, Импульс, Низведение и т.п.), до перевода во что-либо игровое.
4. **Отбор из внешнего массива источников** (теософия, Каббала, Юнг, кельтика, ИВДИВО-синтез и др.) — производится только после того, как архитектура, философия и базовые практики уже заданы; отбор ищет резонанс с уже выстроенным, а не наоборот.
5. **Активация в механику** — переход элемента в игровое правило возможен только после прохождения этапов 1–4 и только через пятиосевой фильтр (DEC-006).

Элемент, не прошедший этапы 1–4, не может быть активирован в механику напрямую. Отсутствие числовых/механических решений на этапах 1–4 — ожидаемое, а не проблемное состояние работы.
