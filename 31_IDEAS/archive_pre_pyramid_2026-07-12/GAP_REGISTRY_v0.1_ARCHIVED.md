## A. Философия → механика (доказательства, которых пока нет)

Это прямое наследие разбора «философия подтверждается / существует только в лоре / противоречит / не переведена в действие».

| # | Что отсутствует | Почему критично | Куда | Статус |
|---|---|---|---|---|
| A1 | Механика **утраты при переходе уровня** (не только системная, но и персональная) | Без неё «вспоминание» ничем не отличается от обычного level-up | GitHub `docs/GAME_MECHANICS.md`, новый раздел | 🔴 |
| A2 | Правило **самостоятельного распознавания конъюнкции** (без автоматического сообщения системы) | Иначе Grand Conjunction — чистая арифметика без азарта (см. математический разбор) | GitHub `docs/GAME_MECHANICS.md` + физический компонент в CARD-001 | 🔴 |
| A3 | Переопределение **deckbuilding** как «раскрытие фиксированной колоды», а не аддитивное накопление | Текущий (предполагаемый) аддитивный deckbuilding прямо противоречит философии | GitHub `docs/GAME_MECHANICS.md` | 🔴 |
| A4 | Переименование/переопределение **«открытия» миров** → «совмещение» с диском, физически присутствующим на столе с начала партии | Слово и, вероятно, механика сейчас означают приобретение новой территории, не раскрытие существующей | GitHub `docs/WORLD_ARCHITECTURE.md` | 🟡 |
| A5 | Механическое определение **«стяжания»** — либо переименовать, либо привязать к раскрытию уже имеющегося, а не к получению нового | Слово семантически противоречит философии в текущем виде | GitHub `docs/GAME_MECHANICS.md` | 🔴 |
| A6 | Обратимость **посвящений** (не одноразовый гейт, а отзываемый статус) | Иначе посвящение = скрытый level-up с религиозной лексикой | GitHub `docs/ENTITIES_AND_CHARACTERS.md` | 🟡 |
| A7 | **Статусы** как позиционные, а не накопительные (могут идти вниз так же легко, как вверх) | Монотонная шкала противоречит идее, что забывание — часть цикла | GitHub `docs/GAME_MECHANICS.md` | 🟡 |
| A8 | **Полномочия** ролей как контекстные/временные, а не постоянные атрибуты | Иначе роль = набор перманентных баффов, не «способность заметить» | GitHub `docs/ENTITIES_AND_CHARACTERS.md` | 🟡 |
| A9 | Механика **состояния забвения** как стартового условия партии (что именно теряет игрок в начале, что предстоит вспомнить) | Без этого нечего механически «вспоминать» — сейчас это утверждение только в тексте | GitHub `docs/NARRATIVE_SYSTEM.md` | 🔴 |
