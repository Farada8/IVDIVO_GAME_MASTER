## `CONTRIBUTING.md` (проект)

```markdown
# Правила ведения документации IVDIVO GAME MASTER

## Статусы документов
Draft → Review → Approved → Deprecated

## Кто меняет статус
- Draft → Review: любой участник (включая Claude/GPT при явном запросе)
- Review → Approved: ТОЛЬКО основатель. Ни одна модель не присваивает Approved самостоятельно.
- → Deprecated: основатель, с указанием документа-замены

## Единый источник номеров решений (DEC-XXX)
Все номера DEC присваиваются ТОЛЬКО последовательно в GitHub DECISIONS.md.
Google Drive XLSX не является источником номеров — только зеркалом для чтения.
Это правило введено после коллизии нумерации DEC-001–007 между Drive и GitHub.

## Терминологический файрвол
Перед мержем любого файла в docs/levels/, docs/characters/, docs/antagonists/
обязательна сверка с glossary/TERMINOLOGY_FIREWALL.md.

## Новые ядровые элементы
Любой файл, вводящий новый уровень, новую роль или новую фундаментальную
механику, не мержится без явной ссылки на номер DEC, утверждённый основателем.
```
