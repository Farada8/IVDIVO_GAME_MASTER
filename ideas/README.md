# ideas — идеи и гипотезы

Этот раздел нужен, чтобы не смешивать новые идеи с утверждённой документацией.

## Правило

Идея не считается решением, пока основатель не утвердил её и она не перенесена в `docs/` или `DECISIONS.md`.

## Разделы

- `mechanics/` — идеи механик.
- `worlds/` — идеи миров и законов миров.
- `monsters/` — существа, угрозы, боссы.
- `cards/` — идеи карт.
- `stories/` — сцены, кампании, сюжетные ветки.
- `art/` — визуальные идеи.
- `business/` — коммерческие идеи.
- `apps/` — цифровые и мобильные идеи.
- `someday/` — сильные идеи не для текущей версии.

## Процесс

1. Сырая мысль попадает в `inbox/`.
2. Рабочая идея переносится в `ideas/`.
3. Проверенная идея оформляется как документ в `docs/`.
4. Утверждённое решение фиксируется в `DECISIONS.md`.
