В разговоре были зафиксированы два значимых блока разработки:

1. Рабочий стандарт единой терминологии игровой вселенной: разделение понятий на канонический / источник-слой, дизайнерско-механический слой и player-facing / коммуникационный слой; запрет подменять термины ИВДИВО словами из фэнтези, RPG, оккультизма или маркетингового упрощения; введение статусов терминов.

## Рабочие разработки

### 1. Единая терминология

Создан рабочий стандарт `UNIFIED_TERMINOLOGY_STANDARD_v0.1_DRAFT`.

Ключевые положения:

- одно понятие — один термин — одно устойчивое значение;
- термин должен иметь статус: CANON, APPROVED_GAME, WORKING, IDEA, REJECTED или TO_VERIFY;
- канонические термины ИВДИВО не переименовываются внутри источникового слоя;
- коммерческий продукт получает самостоятельные player-facing названия;
- игровые термины не должны менять значение канонических понятий;
- маркетинговые термины могут объяснять проще, но не искажать смысл.

Зафиксированные рабочие группы терминов:

- сущности пространства: игровая вселенная, мир, орбиталь, сфера, локация, узел;
- субъекты: игрок, игровой представитель / герой, персонаж, архетип, фракция, сила противодействия;
- процессы: стяжание, синтез, явление, развёртывание, воплощение, преображение, переход, интеграция, активация;
- ресурсы и объекты: Огонь, энергия, импульс, Инструмент, Форма, здание, дар, ключ;
- структура партии: партия, сценарий, цикл, раунд, ход, фаза, действие, реакция, эффект;
- типы карт: персонаж, архетип, мир/орбиталь, локация, Огонь, Инструмент, Форма, здание, испытание, событие, решение, преображение, переход.

### 2. Коммерческая модель франшизы

Создана новая версия `COMMERCIAL_FRANCHISE_MODEL_v0.2_DRAFT`, так как ранее в GitHub была только частичная миграция v0.1.

Основная рабочая формула:

Kickstarter запускает продукт → настольная игра формирует аудиторию → сертифицированные ведущие создают события → клубы создают сообщества → дополнения обеспечивают повторные продажи → цифровая платформа объединяет сеть → мастер-франшизы масштабируют бренд по странам.

Базовые уровни франшизы:

1. Сертифицированный ведущий.
2. Локальный игровой клуб.
3. Городской партнёр.
4. Региональный оператор.
5. Мастер-франшиза страны.

## Новые идеи

- Создать главный словарь игры на 150–250 терминов с полями: термин, слой, статус, определение, механическая функция, player-facing вариант, английский эквивалент, запретные синонимы, связанные документы.
- Ввести для каждого AI-агента обязательный терминологический фильтр: не создавать новые синонимы, если термин уже существует.
- Использовать трёхслойную систему терминов: внутренний источник → дизайнерский термин → публичный термин игрока.
- Развивать франшизу через пилотную тройку: сертифицированный ведущий → локальный клуб → территориальная мастер-франшиза.

## Отклонённые или заменённые варианты

- [SUPERSEDED] Восьмимировая вертикаль как актуальная космологическая архитектура заменена в Google Drive контексте v1.6 на пять вертикальных миров + орбитали параллельных миров. Требуется синхронизация GitHub.
- [REJECTED] Прямая внешняя подача продукта как «игра ИВДИВО» заменена моделью самостоятельной массовой IP-вселенной.
- [REJECTED] Терминологические подмены: Огонь = мана/энергия/магия; Стяжание = лут/покупка; Синтез = крафт/смешивание; Аватар Синтеза = игровой аватар.
- [REJECTED] Франшиза как простое право продавать коробку заменена моделью экосистемы: продукт + события + ведущие + клубы + платформа + лицензии.
