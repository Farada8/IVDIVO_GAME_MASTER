## Краткое резюме

В разговоре была разработана и сохранена системная модель сюжетов для массовой настольной игры и IP-вселенной IVDIVO/LIMINAL. Главный вывод: сюжет не должен быть линейной книгой сценариев, а должен работать как движок состояний, выборов и последствий. Система связывает хоррор-вход, восемь миров, порталы, персонажные дуги, кампанию и мета-сюжет IP.

Основной тематический документ уже существует как `STORY_SYSTEM_v0.1_DRAFT.md`; текущий архив не заменяет его, а фиксирует разговор, статусы и открытые конфликты.

## Утверждённые решения

1. [CANON] Проект создаётся не как эзотерическое пособие и не как «игра ИВДИВО» для внутреннего сообщества, а как массовая международная настольная игра и самостоятельная IP-вселенная.
2. [CANON] Хоррор и dark fantasy остаются коммерческим входом в продукт.
3. [CANON] Масштаб кампании растёт от личного выживания до преобразования и сотворения миров.
4. [CANON] Восемь миров являются центральной вертикальной архитектурой вселенной, механики, кампании и франшизы.
5. [CANON] Механическое ядро синтезирует deckbuilding, engine building, асимметрию, кампании, порталы, ресурсную экономику и изменение миров.
6. [CANON] Внешний язык коммерческой игры должен быть самостоятельным.
7. [CANON] Базовая коробка должна быть законченной игрой; дополнения расширяют продукт, но не чинят ядро.
8. [CANON] Каждая механика должна одновременно усиливать удовольствие, эмоцию, мир, реиграбельность, цифровое масштабирование и франшизный потенциал.
9. [CANON] Сюжет должен возникать из выбора и последствий, а не тормозить игру длинным текстом.
10. [CANON] Каждый значимый результат должен сохраняться в Google Drive и GitHub согласно активному протоколу проекта.

## Рабочие разработки

1. [WORKING] Сюжетная система как движок состояний, событий и последствий.
2. [WORKING] Базовая сюжетная формула: Нарушение → Вход в угрозу → Исследование → Цена → Выбор → Изменение мира → Последствие.
3. [WORKING] Расширенная формула сценария: Мир + Нарушение + Носитель угрозы + Скрытая причина + Цель + Цена + Последствие.
4. [WORKING] Игрок проходит три позиции: выживающий участник → автор решений → преобразователь реальности.
5. [WORKING] Один сценарий меняет минимум два состояния: персонажа, территории, фракции, портала, мира, угрозы или закона реальности.
6. [WORKING] Ветвление определяется игровыми параметрами, а не только выбором параграфа.
7. [WORKING] Пять масштабов сюжета: сцена, миссия, дуга персонажа, кампания, мета-сюжет IP.
8. [WORKING] Фазы кампании: выживание, убежище, территория, связанные миры, преобразование, сотворение.
9. [WORKING] Восемь миров трактуются как разные типы причинности и сюжетного вмешательства.
10. [WORKING] Физическая победа может завершить миссию, но не устранить угрозу кампании, если причина сохраняется на другом уровне.
11. [WORKING] Состояния персонажа: целостный / раненый / заражённый / расколотый / преобразованный.
12. [WORKING] Состояния территории: стабильная / истощённая / заражённая / захваченная / восстановленная.
13. [WORKING] Состояния фракции: союзная / нейтральная / подозревающая / враждебная / расколотая.
14. [WORKING] Состояния портала: закрытый / нестабильный / открытый / контролируемый / заражённый / разрушенный.
15. [WORKING] Состояния угрозы: скрытая / активная / ослабленная / преобразованная / уничтоженная физически / устранённая причинно.
16. [WORKING] Состояния мира: устойчивый / искажённый / распадающийся / восстанавливаемый / преобразованный.
17. [WORKING] Сценарные типы: survival, investigation, portal expedition, faction conflict, territory transformation, false reality, creation objective.
18. [WORKING] Текст сценария выполняет только три функции: эмоциональный вход, изменение правил, фиксация последствия.
19. [WORKING] Персонажная дуга должна менять механику: колоду, действия, экономику, отношения или условия сценария.
20. [WORKING] Базовая кампания предпочтительно resettable; legacy — возможный будущий режим.

## Новые идеи

1. [IDEA] Колоды сюжетного движка: Зовы, Миры, Конфликты, Откровения, Цена, Последствия, Творение.
2. [IDEA] Три режима повествования: приключенческий, психологический, философско-творческий.
3. [IDEA] Метасюжет сезонами: Пробуждение Частей, Война Форм, Раскол Миров, Возвращение Огня, Явление ИВДИВО, Новая Материя.
4. [IDEA] Сюжетные роли игроков: Искатель Огня, Хранитель Формы, Архитектор Миров, Проводник, Исследователь Материи, Защитник Частностей, Вестник ИВО, Преобразователь Искажений.
5. [IDEA] Тестовый vertical slice: убежище + видимая хоррор-угроза + скрытая астральная/ментальная причина + нестабильный портал + две фракции + три финала + одно постоянное последствие.
6. [IDEA] Франшизная связь: регулярные сценарии, ведущие, клубы и цифровая библиотека как часть коммерческой модели.

## Отклонённые или заменённые варианты

1. [REJECTED-FOR-NOW] Линейная книга сценариев как главная модель → заменено системой состояний и последствий.
2. [REJECTED-FOR-NOW] Ветвление только через выбор А/Б → заменено параметрическим ветвлением.
3. [REJECTED-FOR-NOW] Энциклопедический лор вместо игровых событий → заменено требованием проявлять лор через действия и состояния.
4. [REJECTED-FOR-NOW] Прямой player-facing язык ИВДИВО → заменено самостоятельной коммерческой терминологией IP.
5. [REJECTED-FOR-NOW] Сотни сюжетных карт до проверки core loop → заменено фокусом на GAME_CORE и бумажный прототип.
6. [REJECTED-FOR-NOW] Обязательная legacy-кампания в базе → заменено resettable-кампанией для MVP.
7. [REJECTED-FOR-NOW] Отдельная мини-игра для каждого сценария → заменено единым core loop с переменными условиями.

## Связи с другими системами проекта

- `GAME_CORE`: сюжетная система заблокирована до работающего ядра.
- `WORLD_ARCHITECTURE`: восемь миров дают типы причинности и сюжетных вмешательств.
- `GAME_MECHANICS`: порталы, состояния, ресурсы и изменение законов зоны должны быть механическими носителями сюжета.
- `ENTITIES_AND_CHARACTERS`: персонажные дуги зависят от канона и архетипов.
- `DIGITAL_COMPANION`: может хранить флаги кампании, но базовая коробка не должна требовать приложение.
- `COMMERCIAL_FRANCHISE_MODEL`: регулярные сценарии и ведущие поддерживают франшизную модель.
