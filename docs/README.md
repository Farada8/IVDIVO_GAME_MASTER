# docs — рабочие документы проекта

Этот раздел хранит утверждаемые и рабочие документы проекта.

## Структура

- `00_PROJECT_CORE/` — паспорт проекта, правила, индексы, решения.
- `01_RESEARCH/` — рынок, аналоги, Kickstarter, BGG, Reddit, YouTube, отзывы.
- `02_GAME_CONCEPT/` — продуктовая концепция, аудитория, позиционирование.
- `03_MECHANICS/` — core loop, действия, ресурсы, экономика, порталы, баланс.
- `04_CARDS_AND_COMPONENTS/` — карты, жетоны, поля, планшеты, предметы, здания.
- `05_WORLDS/` — восемь миров, законы миров, угрозы, слои реальности.
- `06_CHARACTERS_AND_ARCHETYPES/` — персонажи, роли, архетипы, фракции.
- `07_CAMPAIGNS_AND_STORIES/` — сценарии, кампании, ветвления, последствия.
- `08_VISUAL_BRAND/` — визуальный код, арт-дирекшн, упаковка.
- `09_BUSINESS_MODEL/` — бизнес-модель, монетизация, юнит-экономика.
- `10_KICKSTARTER/` — краудфандинг, pledge tiers, stretch goals.
- `11_DIGITAL_PRODUCT/` — приложение, цифровая версия, платформа.
- `12_FRANCHISE/` — клубы, ведущие, лицензирование, франчайзинг.
- `13_AI_RESULTS/` — промты и результаты GPT, Claude, Gemini, NotebookLM.
- `14_TESTING_AND_BALANCE/` — плейтесты, метрики, отчёты, изменения.
- `15_EDUCATION_AND_PSYCHOLOGY/` — образовательные и психологические версии после доказательства игры.
- `99_ARCHIVE/` — устаревшие и заменённые материалы.

## Правило

В `docs/` попадает только материал с самостоятельной ценностью. Сырые мысли сначала помещаются в `inbox/` или `ideas/`.

Google Drive остаётся рабочей базой и местом первичного редактирования. GitHub используется как Markdown-копия для версионирования, структуры, Codex/Claude и дальнейшей разработки.
