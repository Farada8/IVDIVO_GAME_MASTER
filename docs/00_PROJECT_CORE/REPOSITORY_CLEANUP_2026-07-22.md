## 1. Зачем создан этот документ

В репозитории одновременно находились несколько несовместимых слоёв проекта:

- старый слой: horror как основной вход, восемь миров, `Луч` как центральный ресурс, `Убежище`, ранние 4 роли и старые папки `docs/03_MECHANICS`, `docs/05_WORLDS`, `docs/06_CHARACTERS_AND_ARCHETYPES`;
- новый слой: Dark Cosmic Adventure, пять ярусов + орбитали, 16 ролей, ресурсная тройка Дух / Свет / Огонь, Экополис IVDIVO, кольца / кубик / секторы / аттестации, активная работа в `docs/06_MECHANICS` и `docs/04_CARDS`.

Этот документ фиксирует уборку: старый слой не удаляется физически, но теряет статус активной модели.

## 2. Прямое решение основателя текущей сессии

Основатель дал команду:

```text
убери старьё, что уже обговорено и до сих пор сбои
```

Интерпретация в рамках governance проекта:

- не удалять исторические материалы без необходимости;
- явно отменить устаревшие APPROVED-решения новыми DEC-записями;
- обновить registry, core context, roadmap и open questions;
- пометить старые модели как `SUPERSEDED / OLD MODEL / DO NOT USE AS ACTIVE DESIGN`;
- сохранить полезные элементы старых документов только как источник идей, а не как текущую архитектуру.

6. Для архива старых идей: документы со статусом `SUPERSEDED`, `OLD MODEL`, `DEPRECATED`, `ARCHIVE`.

## 5. Что больше не использовать как активную модель

Не использовать как текущий канон:

- horror как главный жанр проекта;
- восемь вертикальных миров как позвоночник игры;
- схему `4 сферы -> 16 миров -> 64 уровня -> 1024 уровня` как архитектуру;
- `Луч` как единственный / главный ресурс;
- `Убежище` как основной термин базы игроков;
- старые 4 роли как текущий состав ролей;
- registry v1.2 как достоверный список реальных файлов.

## 6. Что сохраняется из старого слоя

Старые материалы могут использоваться как:

- банк идей;
- источник эмоциональных механик;
- источник отдельных карт, угроз, образов, компонентов;
- исторический контекст решений;
- сырьё для переработки в активную модель.

Но они не могут автоматически отменять DEC-018..022.
