## 0. Назначение документа

Этот документ фиксирует рабочую рамку полноценной Kickstarter Core Box для настольной игры IVDIVO. Документ не является финальной производственной сметой и не утверждает окончательный состав коробки. Его задача — задать целевой масштаб продукта, структуру компонентов, объём физических карт, границы stretch goals и expansions, а также правила контроля сложности.

Документ исходит из решения изменить центральное позиционирование: IVDIVO больше не строится как horror / dark fantasy продукт. Рабочий жанр для Kickstarter:

**Cooperative Cosmic Adventure + Mythic Cosmology Campaign Game + Deckbuilding + Portal System + World Transformation.**

## 1. Исходная цель Kickstarter Core Box

Целевая версия продукта:

- полноценный Kickstarter Core Box;
- 1–5 игроков;
- 6-й игрок допускается только как advanced / expansion / team mode после тестов;
- длительность партии: 90–120 минут;
- кампания: 10–12 сценариев;
- 8 персонажей;
- 5 ролей;
- 450–550 физических карт;
- банк разработки: 800–1,000 карт-слотов;
- жанр: cooperative cosmic adventure / mythic cosmology campaign game;
- no app required;
- без обязательных дорогих миниатюр в базовой коробке;
- акцент: исследование миров, порталы, раскрытие причин, Разломы Реальности, выбор Разрыв / Преобразование, последствия кампании.

## 2. Почему маленькая версия 100–140 карт недостаточна

Версия на 100–140 карт подходит для компактного тестового прототипа или малой retail-коробки. Она полезна для проверки базового цикла, но недостаточна для той коммерческой цели, которую сейчас выбирает проект.

Причины:

1. **Недостаточный масштаб кампании.** Для 10–12 сценариев, 8 персонажей и 5 ролей 100–140 карт быстро приведут к повторяемости событий, зон, целей и кризисов.
2. **Недостаточная реиграбельность.** Большая кампанийная игра должна давать разные связки персонажей, карт развития, порталов, Разломов, Корней и последствий. Малый пул карт создаёт ощущение демо-версии, а не большой коробки.
3. **Слабое Kickstarter-впечатление.** Для большой кампании с мифологической космологией, мирами, порталами и трансформацией реальности покупатель ожидает ощущение наполненной коробки: много карт, сценарии, модули мира, заметный прогресс, множество решений.
4. **Невозможность раскрыть 5 ролей.** Если каждая роль должна иметь стартовую колоду, развитие, личные цели и уникальные способы взаимодействия с Корнями, малый объём карт обрежет асимметрию.
5. **Недостаточный объём для системы Разлом → Проявление → Искажение → Корень.** Эта система требует отдельного пула карт для каждого слоя. Если сделать слишком мало карт, ядро игры будет выглядеть богато в тексте, но бедно на столе.
6. **Недостаточно места для последствий кампании.** Шрамы, преобразованные зоны, союзники, изменения Убежища и развилки требуют отдельного физического или книжного носителя.

Итог: 100–140 карт оставляются как диапазон для раннего прототипа v0.1 / компактного варианта, но не как целевой масштаб IVDIVO Core Box.

## 4. Что уходит в stretch goals

Stretch goals должны усиливать базовую коробку, но не ломать её завершённость. База должна быть законченной игрой.

Подходящие stretch goals:

- дополнительные персонажи;
- дополнительные карты развития;
- новые Инструменты;
- новые Формы;
- дополнительные зоны;
- дополнительные Разломы / Корни;
- дополнительные союзники и жители;
- альтернативные сценарии;
- улучшенные компоненты: linen finish cards, thicker boards, dual-layer player boards;
- acrylic standees вместо картонных standees;
- дополнительные player aid cards;
- мини-артбук;
- soundtrack / digital lore booklet как необязательный бонус;
- улучшенный органайзер;
- solo challenge pack;
- advanced 6-player mode, только если тесты подтвердят управляемый downtime.

Не использовать как stretch goals:

- эксклюзивные карты, без которых база становится неполной;
- обязательное приложение;
- десятки новых правил поверх старых;
- дорогие миниатюры, резко увеличивающие себестоимость и риски доставки;
- неконтролируемое раздувание кампании до 20+ сценариев в первой коробке.

## 5. Что уходит в expansions

Expansions должны открывать новые горизонты после доказательства core loop, а не перегружать первую коробку.

Подходящие expansions:

1. Новые вертикальные миры как полноценные большие модули.
2. Орбитали: Ад, Рай, Аидия, Демонский мир, Омарный мир и будущие параллельные миры.
3. Новые кампании на 6–10 сценариев.
4. Новые роли.
5. Фракционный режим.
6. Соревновательный или semi-coop режим.
7. Большой модуль Созидания миров.
8. Цифровой companion после релиза, если есть спрос.
9. Дуэльная / compact карточная линия.
10. Отдельная образовательная / фасилитируемая линия, но не как базовая игра.

В первой Core Box нельзя пытаться показать всю космологию. Core Box должен дать завершённую первую кампанию и понятное обещание: открыть порталы, исследовать миры, найти Корни Разломов и решить судьбу реальности через Разрыв или Преобразование.

## 6. Какие компоненты должны быть картами

Картами должны быть те элементы, которые требуют перемешивания, скрытия, случайного появления, выбора из рынка, обновления, кампанийного открытия или замены между партиями.

Картами должны быть:

- стартовые карты игроков;
- карты развития;
- Инструменты;
- Формы;
- зоны, если они двусторонние и модульные;
- порталы;
- режимы Вихря;
- риски перехода;
- Разломы;
- Проявления;
- Искажения;
- Корни;
- Разрыв;
- Преобразование;
- жители;
- союзники;
- посредники;
- события раунда;
- испытания;
- цели;
- последствия;
- Шрамы;
- преобразованные зоны;

- карты помощи.

## 7. Какие компоненты лучше сделать планшетами, жетонами или книгой сценариев

Не всё нужно превращать в карты. Иначе игра станет тяжёлой в производстве и медленной в использовании.

Планшетами лучше сделать:

- персонажей;
- роли, если роль не должна перемешиваться;
- Убежище;
- Вихрь;
- трек Накала;
- общую карту кампании / состояние миров;
- reference board для фаз раунда.

Жетонами лучше сделать:

- Огонь;
- Тень;
- Накал;
- Импульс;
- Якоря;
- повреждения;
- заражение портала;
- Шрамы, если они простые;
- маркеры персонажей;
- маркеры контроля зоны;
- маркеры стабильная / искажённая / преобразованная.

Книгой сценариев лучше сделать:

- длинный сюжетный текст;
- setup каждого сценария;
- специальные правила сценария;
- таблицы переходов;
- альтернативные финалы;
- подсказки обучения;
- часть законов мира, если они не требуют случайного появления;
- часть общих целей, если они привязаны к конкретному сценарию.

Campaign Log лучше сделать отдельной тетрадью / листами, а не сотнями мелких карт, если эффект легко записать.

## 10. Риски большой коробки

1. **Перегруз правилами.** Если каждая колода вводит отдельное мини-правило, игроки не смогут удержать систему.
2. **Downtime при 5 игроках.** Чем больше игроков, тем важнее одновременное планирование, быстрые ходы и ограничение анализа.
3. **Рост себестоимости.** 500 карт, книги, жетоны, планшеты и органайзер требуют ранней производственной проверки.
4. **Риск повторения эффектов.** Большой банк карт легко превращается в десятки одинаковых эффектов с разными названиями.
5. **Риск лор-перегруза.** Космология должна проявляться через действия, а не через длинные тексты.
6. **Баланс ролей.** 5 ролей должны быть полезны во всех сценариях, но не обязательны каждая по отдельности.
7. **Риск слишком долгой партии.** Цель 90–120 минут требует строгого контроля числа фаз, раундов и действий.
8. **Риск “большой, но пустой” коробки.** Количество карт не должно быть самоцелью. Каждая карта должна давать решение, изменение состояния или кампанийное последствие.
9. **Риск слишком раннего раскрытия всей космологии.** Core Box должен показать масштаб, но не выложить сразу все миры, орбитали и высшие системы.
10. **Риск производственного провала stretch goals.** Stretch goals должны улучшать ценность, а не взрывать себестоимость и сроки доставки.

## 11. Как удержать сложность под контролем

1. **Один главный цикл партии.** Все карты должны обслуживать один цикл: персонаж → роль → колода → зона → портал → Разлом → Искажение → Корень → Разрыв / Преобразование → последствие → кампания.
2. **Не более 5 основных ресурсов в базовой коробке.** Рекомендуемые базовые ресурсы / треки: Огонь, Тень, Накал, Импульс, Якорь / Связь как ограниченный маркер.
3. **Одна структура кризиса.** Разные кризисы должны использовать одну понятную модель: Разлом → Проявление → Искажение → Корень.
4. **Одна структура финального выбора.** Каждый Корень должен решаться через Разрыв или Преобразование.
5. **Сценарии вводят правила постепенно.** Сценарий 1: базовый ход, зона, ресурс. Сценарий 2: портал. Сценарий 3: Разлом. Сценарий 4: Искажение. Сценарий 5: Корень. Сценарий 6: Разрыв / Преобразование. Сценарий 7+: орбитальное вмешательство и более сложные законы.
6. **Карты должны иметь короткий игровой текст.** Цель: большинство карт читаются за 5–10 секунд. Сложный лор — в книге сценариев, не на каждой карте.
7. **Дубли удаляются без жалости.** Если две карты дают одинаковый выбор, остаётся только лучшая.
8. **Каждая карта получает статус.** DRAFT, TEST, KEEP, CUT, REWORK, EXPANSION, STRETCH_GOAL.
9. **Плейтесты решают состав, а не вкус автора.** Карта остаётся в Core Box только если она создаёт решение, понятна игроку, не ломает баланс, усиливает жанр cosmic adventure и связана с одной из ключевых систем.
10. **Core Box не обязан содержать всю IVDIVO-вселенную.** Первая коробка должна быть входом в IP, а не энциклопедией. Остальные миры, орбитали, фракции, режимы и глубокие системы разворачиваются через stretch goals и expansions.

## 12. Производственное решение v0.1

Рабочее решение:

**IVDIVO Core Box проектируется как большая Kickstarter-коробка на 1–5 игроков, 90–120 минут, 10–12 сценариев, 8 персонажей, 5 ролей и 450–550 физических карт.**

Банк разработки должен содержать 800–1,000 карт-слотов, из которых после тестов выбирается финальный состав Core Box.

Малый диапазон 100–140 карт сохраняется только как прототипная и/или компактная версия, но не как целевая Kickstarter Core Box.

Статус решения: **DRAFT / REQUIRES PLAYTEST + PRODUCTION QUOTE.**
