## 1. Core business thesis

Portals should not be treated only as one printed board game.

The stronger business model is:

```text
Family board game -> Kickstarter product -> expansions -> community events -> clubs/demo tables -> digital companion -> educational/family programs -> IP universe/franchise.
```

The box is the entry point. The wider business is the product ecosystem.

## 2. Market position

Public-facing positioning:

```text
Portals is a colorful family fantasy adventure board game where players collect magical Sparks, open portals, recruit heroes, face creatures, and race through three rings of adventure to unlock the Great Portal.
```

Do not lead with metaphysical complexity. Lead with immediate table fun.

Internal IVDIVO depth can exist underneath the family product.

## 3. First product

The first commercial product should be a tight family adventure box.

Recommended box:

```text
Title: Portals
Players: 2-6
Age: 8+
Time: 30-45 minutes
Core hook: collect Sparks, open portals, race to the inner source
Board: 3-ring circular board
Cards: 100-140
Components: Sparks, pawns, portal markers, 2 dice, character cards
```

Avoid heavy miniatures in edition 1.

## 4. Business stages

### Stage 0: Validation prototype

Goal: prove the game is fun.

Tasks:

```text
- print cheap PNP prototype;
- run 10-20 playtests;
- cut boring rules;
- test Sparks economy;
- test portal market row;
- test 6 character powers;
- record feedback.
```

Budget: EUR 50-300.

### Stage 1: Prelaunch prototype

Goal: make people want to follow the project.

Tasks:

```text
- produce 3-5 nice copies;
- create 2-page quick rules;
- create 15-30 polished card illustrations;
- photograph components;
- film short gameplay moments;
- launch landing page;
- collect emails;
- start TikTok/YouTube Shorts/Instagram/Reels.
```

Budget: EUR 500-3000 depending on outsourcing.

### Stage 2: Kickstarter launch package

Goal: raise USD 30k-50k.

Required:

```text
- 5-10 reviewer/demo copies;
- campaign video;
- full rulebook draft;
- playthrough video;
- manufacturing quotes;
- shipping estimates;
- pledge levels;
- stretch goal plan;
- 500-3000 qualified email leads;
- 5-10 reviewer/previewer contacts.
```

Budget: EUR 3000-10000 depending on quality level.

### Stage 3: Fulfillment and first retail proof

Goal: deliver successfully and create trust.

Tasks:

```text
- manufacture only after final quote;
- avoid overcomplicated stretch goals;
- handle shipping conservatively;
- collect reviews;
- open direct store after fulfillment;
- sell remaining stock through website and local stores.
```

### Stage 4: Expansion system

After successful delivery, create expansion lines:

```text
- new portal worlds;
- new characters;
- new adventure decks;
- seasonal event packs;
- family/co-op expansion;
- advanced strategic expansion;
- IVDIVO lore expansion for deeper audience.
```

## 6. Recommended first business model

The first realistic path:

```text
1. Build playable prototype.
2. Run local playtests in Ireland.
3. Build small online audience.
4. Launch Kickstarter for base game.
5. Deliver successfully.
6. Sell direct online.
7. Add expansions and event kits.
8. Develop club/host/franchise layer after proof.
```

Do not start with the franchise first. Prove the game first.

## 7. Funding goal logic

A USD 30k-50k Kickstarter likely needs roughly:

```text
600 backers x USD 50 average pledge = USD 30,000
800 backers x USD 62 average pledge = about USD 50,000
1000 backers x USD 50 average pledge = USD 50,000
```

Therefore the real business problem is not only art or rules. It is audience conversion.

## 8. Cost structure

Early cost categories:

```text
- prototype printing;
- card/board layout;
- illustration;
- video and photography;
- website/landing page;
- email software;
- ads/testing;
- reviewer copies;
- manufacturing sample;
- shipping estimates;
- legal/IP cleanup.
```

The first edition should avoid expensive components that destroy margin:

```text
- too many miniatures;
- oversized custom plastic;
- heavy box;
- too many punchboards;
- excessive card count;
- too many stretch goals.
```

## 9. Unit economics target

For a pledge around USD/EUR 49-69:

```text
Manufacturing cost target: 20-30% of pledge.
Platform/payment fees: about 8-10%.
Shipping should be charged separately where possible.
Reserve/contingency: at least 10-15%.
```

If the game cannot be manufactured and fulfilled safely inside these limits, reduce components.

## 10. Marketing strategy

### Audience segments

```text
- families who like colorful adventure games;
- casual board gamers;
- parents buying smart family games;
- fantasy fans;
- Kickstarter tabletop collectors;
- teachers/community organizers later;
- IVDIVO/lore audience later, not first.
```

### Main message

```text
Open portals. Collect Sparks. Race through three rings of adventure. Unlock the Great Portal.
```

### Content to publish

```text
- short videos showing Sparks and portals;
- before/after board evolution;
- character reveal posts;
- card market tension examples;
- 30-second turn examples;
- playtest reactions;
- prototype table photos;
- Kickstarter countdown.
```

## 11. Local Ireland strategy

Use Ireland as test market, not final limit.

Actions:

```text
- test with families and local board game groups;
- run demo nights in Dublin/Galway/Cork if possible;
- contact local game shops for feedback;
- attend small conventions/markets;
- build mailing list from local testers;
- use testimonials and photos for Kickstarter page.
```

## 12. Retail strategy after Kickstarter

After fulfillment:

```text
- direct website sales first;
- local game shops on consignment or small wholesale;
- Amazon only later if margin allows;
- convention/event sales;
- bundles with expansion packs.
```

Retail needs lower wholesale price, so Kickstarter/direct sales are better early.

## 14. Franchise layer

Franchise should come after product proof.

Potential layers:

```text
- certified host kit;
- school/family event kit;
- club kit with multiple copies;
- local tournament pack;
- translated regional license;
- digital scenario subscription.
```

This matches the wider IVDIVO franchise model, but should not be launched before the base game has traction.

## 15. Main risks

```text
- too many mechanics;
- unclear audience;
- beautiful but untested prototype;
- manufacturing cost too high;
- weak prelaunch list;
- no reviewer proof;
- confusing metaphysical positioning;
- no delivery plan;
- overpromising expansions before base game.
```

## 16. Main opportunities

```text
- Sparks can become iconic components;
- three-ring board has clear table presence;
- expansions are natural;
```

```text

Build a proof pipeline:
Prototype -> playtest -> audience -> Kickstarter -> fulfillment -> expansions -> events -> franchise.
```
