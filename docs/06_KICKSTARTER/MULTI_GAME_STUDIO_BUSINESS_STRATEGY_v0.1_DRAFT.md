## 1. Founder correction

```text
The goal is not only one board game. The goal is many different games and a business built around a pipeline of tabletop products.
```

## 2. Core thesis

The business should be designed as a small tabletop studio / game factory, not as a single title.

Recommended model:

```text
Idea portfolio -> fast prototypes -> playtests -> select winners -> Kickstarter/direct launch -> expansions -> family of games -> IP/franchise ecosystem.
```

## 3. Product portfolio logic

The studio should maintain several product lines at different complexity and cost levels:

```text
1. Small card games: cheap, fast, easy to test.
2. Family board games: colorful mainstream products.
3. Strategy/adventure games: deeper hobby products.
4. Party/social games: easier retail/event products.
5. Educational/family development games: schools, parents, facilitators.
6. IVDIVO lore/advanced games: deeper IP layer after market proof.
```

The purpose is not to make everything at once, but to create a pipeline where weak ideas die early and strong ideas get development budget.

## 4. Portfolio roles

Each game must have a business role:

```text
Gateway game: simple, cheap, brings families in.
Hero game: strongest Kickstarter/title product.
Card-game line: low production cost and repeat releases.
Expansion line: monetizes existing audience.
Event game: good for clubs/demo nights.
Educational game: later B2B/channel opportunity.
Digital companion: keeps audience active.
```

## 6. Development pipeline

Every game should pass gates:

```text
Gate 0: one-page concept.
Gate 1: paper prototype in 48 hours.
Gate 2: 5 internal tests.
Gate 3: 10 external tests.
Gate 4: cost estimate and component reduction.
Gate 5: visual prototype.
Gate 6: landing page / audience test.
Gate 7: Kickstarter/direct launch decision.
```

If a game cannot become fun within 10-20 tests, pause it.

## 7. Cost discipline

Start with low-cost formats first:

```text
Card-only games.
Roll-and-write / print-and-play.
Small-box games.
Games reusing the same Spark tokens.
Games sharing art/IP/components.
```

Avoid at early stage:

```text
Miniature-heavy games.
Huge boards.
Too many custom plastics.
Massive campaign games.
Multiple big boxes before proven demand.
```

## 8. Revenue model

```text
Kickstarter launches.
Direct website sales.
Print-and-play packs.
Small boxed card games.
Expansions.
Accessory token sets.
Art prints/books.
Event/demo nights.
Licensing and translations later.
Club/host kits after proof.
```

## 9. Business stages

### Stage 1: Micro-studio validation

```text
Create 3-5 small prototypes.
Test locally in Ireland.
Find which game produces laughter, tension, replay requests.
Build one shared website/brand hub.
```

### Stage 2: First commercial launch

```text
Choose one hero product.
Build prototype, page, email list, reviewers.
Launch Kickstarter or small direct preorder.
```

### Stage 3: Portfolio expansion

```text
Release one small card game or expansion every 6-9 months.
Use one successful audience to sell the next product.
```

### Stage 4: Studio/IP system

```text
Create game families.
Add digital companion.
Sell host kits and local event kits.
Develop IVDIVO deeper lore products.
```

## 10. Recommended 12-month focus

```text
Months 1-2: build and test Portals + 2 small card-game spin-offs.
Months 3-4: choose strongest 1-2 products; stop weak ones.
Months 5-6: create polished prototype and prelaunch funnel for strongest product.
Months 7-9: reviewer outreach, playthrough, quotes, campaign page.
Months 10-12: launch only if audience metrics are strong; otherwise continue testing and list building.
```

## 11. Key opportunity

Many games is safer than one game only if the studio has a selection system.

Wrong model:

```text
Make 20 unfinished games.
```

Right model:

```text
Generate many ideas -> prototype fast -> kill weak ideas -> polish winners -> launch one at a time.
```
