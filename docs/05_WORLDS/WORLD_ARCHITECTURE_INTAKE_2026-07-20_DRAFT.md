## Связь с предыдущими исследованиями

`WORLD_ARCHITECTURE_v0.1_DRAFT` остаётся главным рабочим документом по архитектуре миров. Этот intake-документ не заменяет его и не присваивает статус APPROVED ни одной новой идее.

## [APPROVED] Утверждённые решения, которые ограничивают перенос

1. Восемь миров — центральная вертикальная архитектура проекта.
2. Хоррор является коммерческим входом в продукт.
3. Внешний язык IP должен быть самостоятельным, не прямой терминологией ИВДИВО.
4. Дополнения расширяют базовую игру, но не чинят ядро.
5. Механика должна работать одновременно как игра, эмоция, мир, реиграбельность и масштабирование.

## [HYPOTHESIS] Что можно сохранить из раннего черновика

| Элемент | Возможный перенос | Ограничение |
|---|---|---|
| Частный мир игрока | Tableau / engine-building планшет | Проверить против core loop |
| Формула карты | Внутренний паспорт карты | Переименовать термины для IP |
| Цикл развития | Campaign progression / state machine | Не заменять emotional arc |
| Хоррор как распад формы | Система искажения | Привязать к Инфернальному миру / Reality Fracture |
| Центральный Дом | Hub / убежище / стартовая зона | Не выводить игру сразу в сакральный масштаб |
| Глубины | High-risk dungeon/exploration зона | Решить: зона Физического/Инфернального или отдельный модуль |
| Пути реализации | Фракции/классы/школы колод | Не утверждать названия |
| Частные здания | Категории карт зданий | Проверить экономику и баланс |

## [REJECTED] Что не переносится напрямую

1. Семь уровней как текущая архитектура.
2. Исток как управляемый или центральный игровой объект базовой коробки.
3. Прямой публичный язык ИВДИВО.
4. Ангельские/Лучевые/Синтезные миры как утверждённые названия коммерческого IP.
5. Энциклопедическое раскрытие 64 видов материи в базовой коробке.

## [GAME-ADAPTATION] Предварительная карта переноса

| Ранний элемент | Куда может перейти в 8-мировой архитектуре |
|---|---|
| Миры Искажения и Хоррора | Инфернальный мир; состояние порчи любого мира |
| Проявленные миры | Физический мир; зона ресурсов и строительства |
| Ангельские миры | Мир Души/Синтеза; восстановление, связи, коллективные действия |
| Миры Лучей | Мир Воли/Архетипов; асимметрия ролей и полномочий |
| Подземные миры | Физический/Инфернальный модуль dungeon crawl |
| Миры Высшего Синтеза | Огненный мир или поздняя кампания после проверки ядра |
| Частный мир | Игроковое tableau/убежище/территория |

## Альтернативы

### Альтернатива A — полностью отбросить ранний черновик

Плюс: чистота текущей архитектуры.  
Минус: теряются полезные авторские идеи о Частном мире, искажении, зданиях, фракционных путях.

Отклонено: создаёт конфликт с DEC-003, PROJECT_CORE_CONTEXT и правилом самостоятельного IP-языка.

### Альтернатива C — сохранить как банк идей и сделать mapping

Рекомендовано: идеи не теряются, но не подменяют канон.
