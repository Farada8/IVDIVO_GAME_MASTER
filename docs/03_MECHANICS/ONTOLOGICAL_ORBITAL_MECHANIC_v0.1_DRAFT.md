## 0. Операционное правило для будущих разговоров

Этот документ должен использоваться как рабочая опора при любых следующих разговорах проекта о:

- механике кругов и орбиталей;
- клетках ходов;
- персонажах и инструментах;
- экономике ресурсов;
- порталах / телепортах / переходах между мирами;
- наложении движения орбиталей и миров;
- «химии событий»;
- действиях из Синтезного мира вниз.

Нельзя заново начинать механику с произвольного сюжета или чужой локации. Сначала применять эту модель, затем уточнять её через новые решения основателя.

## 1. Статус

Это не APPROVED-канон и не финальный rulebook. Это рабочая гипотеза механики, которую надо использовать как активную основу проектирования, пока основатель не заменит её другой структурой.

В этом документе есть два слоя:

1. **SOURCE / FOUNDER-INPUT** — прямые решения основателя из ORBITAL_PARALLEL_WORLDS.
2. **DRAFT / HYPOTHESIS** — механическая разработка на базе этих решений.

## 2. Опорные зафиксированные решения

Из PROJECT_CORE_CONTEXT:

- игра должна быть массовой настольной игрой и самостоятельной IP-вселенной;
- формула проекта: сильная настольная механика + глубокая авторская вселенная + архетипическая эмоциональность + коллекционность + многоуровневое развитие + цифровая масштабируемость + франшизный потенциал;
- механическая база включает deckbuilding / engine building, асимметричных персонажей, модульные миры, ресурсную экономику, порталы, контроль маршрутов, изменение законов реальности, последствия между сценариями;
- портал не является простым перемещением: у него есть источник, назначение, условия открытия, ключ/допуск, стоимость, риск, пропускная способность, направление воздействия, последствия, возможность контроля, повреждения, заражения и блокировки;
- механика должна одновременно усиливать игру, эмоцию, мир, стратегический выбор, сюжет, кампанию, дополнения, цифровую версию, коллекционность и сообщество.

Из ORBITAL_PARALLEL_WORLDS:

- основной вертикальный мир, клетки / сектора круга мира и орбитальные параллельные миры — разные сущности;
- орбитальные параллельные миры входят кругами в соответствующий мир и друг в друга;
- игровые эффекты орбиталей ещё не зафиксированы как канон.

## 3. Главная формула механики

```text
МИР = большой круг.
КЛЕТКИ МИРА = места действия внутри круга.
ОРБИТАЛИ = параллельные миры-кольца, входящие в этот мир и друг в друга.
ПЕРСОНАЖ = существо, способное действовать на одном или нескольких уровнях.
ИНСТРУМЕНТ = право или способ менять клетки, орбитали, связи, порталы или законы.
СОБЫТИЕ = реакция от наложения мира + орбитали + клетки + действия + инструмента.
```

## 4. Главный игровой цикл

```text
1. Орбитали вращаются.
2. Игроки перемещаются или действуют на клетках мира.
3. Проверяются совпадения: клетка мира + клетка орбитали + позиция персонажа.
4. Совпадение создаёт событие.
5. Игрок применяет инструмент или ресурс.
6. Событие даёт ресурс, долг, портал, искажение, закон, преобразование или новую форму.
7. Последствие записывается в состояние мира / кампании.
```

## 5. Химия событий

Событие возникает как химическая реакция:

```text
МИР + КЛЕТКА + ОРБИТАЛЬ + ПОЛОЖЕНИЕ ОРБИТАЛИ + ДЕЙСТВИЕ + ИНСТРУМЕНТ = СОБЫТИЕ
```

Примеры рабочих реакций:

```text
Физическая клетка «Потеря» + Рай + действие «помочь» = восстановление через жертву.
Физическая клетка «Ресурс» + Мир фей и лепреконов + действие «взять» = удача с подвохом / малый долг.
Физическая клетка «Конфликт» + Нижний мир + действие «атака» = быстрое решение с искажением.
Тонкая клетка «Сон» + Аштар / Сириус + звёздный инструмент = телепорт / сигнал / маршрут.
Тонкая клетка «Искажение» + Ангельская цивилизация + весть = очищение через раскрытие правды.
Метагалактическая клетка «Аркан» + Орбиталь Архетипов + выбор = новая роль персонажа.
Метагалактическая клетка «Имя» + Орбиталь 99 Имён + закон = разворот судьбы / новый закон кампании.
```

## 6. Типы конъюнкций

```text
Малая конъюнкция = персонаж + клетка мира + одна орбиталь.
Большая конъюнкция = персонаж + клетка мира + две орбитали.
Великая конъюнкция = персонаж + клетка мира + 3 орбитали или верхний мир.
Синтезная конъюнкция = верхний мир + нижний мир + орбиталь + инструмент Синтеза.
```

Эффекты:

- малая конъюнкция создаёт обычное событие;
- большая конъюнкция создаёт выбор с ценой;
- великая конъюнкция меняет состояние зоны, открывает портал или даёт закон;
- синтезная конъюнкция позволяет сотворить новую форму или переписать нижний закон.

## 9. Персонажи — DRAFT

Персонажи не должны быть простыми классами «воин/маг». Они являются онтологическими ролями — существами, у которых есть разные права вмешательства в миры.

### 9.1. Странник Кругов

Роль: движение, разведка, переходы.  
Инструмент: Компас Перехода.  
Сила: видит ближайшее совпадение орбитали и мира.

### 9.2. Архитектор Форм

Роль: строительство, закрепление результата, создание объектов.  
Инструмент: Печать Формы.  
Сила: превращает временный эффект в постоянный объект.

### 9.3. Провидец Орбиталей

Роль: прогноз движения колец.  
Инструмент: Око Согласования.  
Сила: видит будущую конъюнкцию на 1–2 раунда вперёд.

### 9.4. Алхимик Событий

Роль: изменение химии события.  
Инструмент: Реторта Причин.  
Сила: заменяет один компонент реакции, например страх на вдохновение.

### 9.5. Страж Порога

Роль: защита переходов и порталов.  
Инструмент: Ключ Предела.  
Сила: закрывает повреждённый портал или удерживает его открытым.

### 9.6. Носитель Луча

Роль: Тонкий мир, миссия, направление.  
Инструмент: Жезл Луча.  
Сила: усиливает действие, если оно совпадает с миссией персонажа.

### 9.7. Звёздный Навигатор

Роль: Аштар / Сириус / межмировые маршруты.  
Инструмент: Карта Звёздного Маршрута.  
Сила: открывает телепорты через орбитальные совпадения.

### 9.8. Ангельский Посланник

Роль: очищение, восстановление, благословение.  
Инструмент: Книга Вести.  
Сила: переводит негативное состояние в испытание с наградой.

### 9.9. Арканист

Роль: Метагалактический Дух, архетипы, арканы.  
Инструмент: Колода Арканов.  
Сила: временно меняет роль персонажа или закон клетки.

### 9.10. Синтезатор

Роль: верхний мир, соединение, сотворение.  
Инструмент: Столп Синтеза.  
Сила: действует сверху вниз через несколько миров.

Тонкие инструменты: зеркало, сонник, нить, печать связи, жезл луча.

## 12. Порталы и телепорты — DRAFT

Типы переходов:

```text

Тонкий переход: сон, образ, зов, память, связь.

Метагалактический портал: переход через закон, аркан, имя, полномочие.
Огненный разрыв: прожечь путь через невозможное.
Синтезный нисходящий канал: верхняя сила действует вниз.
```

Портал должен проверяться по уже принятой формуле: источник, назначение, условия открытия, допуск, стоимость, риск, пропускная способность, направление воздействия, последствия и возможность контроля / повреждения / блокировки.

## 13. Действия из Синтезного мира вниз — DRAFT

Игрок, действующий из Синтезного мира, не ходит как обычная пешка. Он выбирает линию нисхождения:

```text
Синтезный мир
↓
Огненный переход
↓
Мир Метагалактического Духа
↓
Тонкий мир
↓
Физический мир
```

### 13.1. Столп Прожига

Провести столп сверху вниз через выбранную линию миров.

Эффекты:

- сжечь нижнее искажение;
- разрушить ложный договор;
- открыть чистый канал;
- уничтожить старую форму, если она не выдерживает.

### 13.2. Развёртывание Удачи

Игрок меняет не ресурс, а вероятность / благоприятность событий на нижнем круге.

Эффекты:

- повернуть одну орбиталь на 1 клетку;
- отменить плохое совпадение;
- создать благоприятную конъюнкцию;
- открыть удачный путь через Рай, Фейный мир или Ангельскую цивилизацию.

### 13.3. Перепись Закона

Игрок меняет правило клетки или зоны до конца сценария или кампании.

### 13.4. Сотворение Новой Формы

Из ресурсов разных миров создаётся новый постоянный объект.

Пример:

```text
Материя + Память + Аркан + Огонь + Синтез = Новый Храм Перехода.
```

### 13.5. Снятие Нижнего Договора

Синтезный игрок выбирает нижний долг и поднимает его на верхний суд.

### 13.6. Нисхождение Аватара

Игрок временно создаёт аватар в нижнем мире. Он действует 1–3 раунда, затем оставляет новый закон или шрам.

## 15. Что не утверждено

Пока не утверждены:

- финальные названия персонажей;
- точные числа клеток на каждом круге;
- скорость вращения каждой орбитали;
- баланс ресурсов;
- публичная коммерческая терминология;
- первый сценарий;
- визуальная форма поля;
- связь этой модели с обновлением DECISIONS, DOCUMENT_REGISTRY и CHANGELOG.
