## 1. Прямая фиксация основателя

После первичной идеи:

> думаю на орбиталях 16 ходов, а на основных дисках 64

основатель уточнил:

> это набросок, орбиталь арканов 22

и добавил:

> мы уточним по ходу

## 2. Текущая рабочая формула

```text
Основные диски миров = 64 клетки / хода.
Орбитали = переменный размер.
16 клеток = рабочая базовая размерность для многих орбиталей.
22 клетки = специальная размерность для Орбитали Архетипов / Арканов.
Остальные орбитали уточняются по ходу разработки.
```

## 3. Что меняется относительно MECH-003

Документ `ORBITAL_GRID_64_16_v0.1_FOUNDER_DIRECTION.md` остаётся полезным как технический набросок соотношения 64/16.

Но он НЕ должен читаться как окончательное правило “все орбитали всегда 16 клеток”.

Правильное чтение:

```text
64 = базовый размер основного диска мира.
16 = стандартная / тестовая размерность орбитали.
22 = исключение для Орбитали Арканов.
N = допустимая переменная размерность для других орбиталей после уточнения основателем.
```

## 4. Орбиталь Арканов / Архетипов

Орбиталь Архетипов / Арканов входит в Мир Метагалактического Духа и должна рассматриваться как кандидат на 22 позиции, потому что логика Арканов естественно требует 22 больших архетипических хода.

Рабочая формула:

```text
Орбиталь Архетипов / Арканов = 22 позиции / хода.
```

Статус: FOUNDER-DIRECTION / DRAFT. Полный список 22 позиций требует отдельной разработки.

## 5. Механическое следствие

Если основной диск имеет 64 клетки, а орбиталь Арканов имеет 22 позиции, то простое соотношение 4:1 больше не работает.

Это может быть даже сильнее для игры: несовпадающие циклы создают редкие и ценные конъюнкции.

```text
64-клеточный диск мира + 22-позиционная орбиталь Арканов = длинный цикл совпадений.
```

Игровое значение:

- Аркановые события должны быть реже, чем обычные 16-позиционные орбитальные события.
- Они могут иметь больший вес: смена роли, судьбы, закона, архетипа, кампанийного направления.
- Орбиталь Арканов лучше использовать не как частый эффект, а как высокий механизм судьбы / метароли.

## 6. Общий принцип переменных орбиталей

Каждая орбиталь может получить собственное число клеток, если её внутренняя логика этого требует.

Рабочие варианты:

```text
16 = стандартная орбиталь, удобная для 64-клеточного диска.
22 = аркановая / архетипическая орбиталь.
64 = редкая зеркальная орбиталь, полностью равная основному диску.
99 = возможная полная орбиталь 99 Имён, но не для базового прототипа.
64 = возможная орбиталь 64 Духов, но не для базового прототипа.
10 = возможная орбиталь Сефир, если делать 10 узлов.
12 = возможная ангельская / зодиакальная / циклическая орбиталь, если будет такое решение.
```

Эти числа не утверждены как финальные, кроме текущего направления: основной диск 64, Орбиталь Арканов 22 как отдельный набросок.

## 7. Правило для будущей разработки

При разработке клеток, карт и эффектов нельзя автоматически назначать всем орбиталям 16 клеток.

Правильный порядок:

1. Сначала определить смысл орбитали.
2. Затем определить её естественную числовую структуру.
3. Затем проверить, как она взаимодействует с 64-клеточным основным диском.
4. Затем назначить скорости вращения.
5. Затем разработать карты событий и конъюнкций.

## 8. Для бумажного прототипа

Первый тест можно вести на упрощённой схеме:

```text
Физический диск = 64 клетки.
1–2 физические орбитали = 16 клеток.
Орбиталь Арканов = 22 клетки, как высокий редкий слой.
```

Если 22-позиционная орбиталь перегружает первый тест, её можно временно заменить мини-колодой 22 Арканов без физического диска, но в архитектуре оставить именно как 22-позиционную орбиталь.
