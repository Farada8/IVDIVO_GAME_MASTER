## 1. Прямая фиксация основателя

Основатель предложил новую размерность ходовой сетки:

> думаю на орбиталях 16 ходов, а на основных дисках 64

Рабочая формула:

```text
Основной диск мира = 64 клетки / хода.
Орбитальный параллельный мир = 16 клеток / ходов.
Соотношение = 64 : 16 = 4 : 1.
```

Статус: направление основателя для разработки механики. Не APPROVED-rulebook, пока не протестировано бумажным прототипом.

## 2. Почему это важно

Это заменяет прежнюю малую тестовую сетку `12 клеток основного мира + 6 клеток орбитали`, использованную в ONTOLOGICAL_ORBITAL_MECHANIC_v0.1_DRAFT как быстрый прототип.

Новая сетка лучше подходит для большой игры, потому что:

1. 64 клетки дают полноценный круг мира с глубокой навигацией.
2. 16 клеток орбитали дают компактный, но выразительный цикл параллельного мира.
3. Соотношение 4:1 позволяет каждой орбитальной клетке накрывать / резонировать с четырьмя клетками основного мира.
4. 64 удобно делить на 8 секторов по 8 клеток, 4 четверти по 16 клеток или 16 малых дуг по 4 клетки.
5. 16 удобно делить на 4 фазы по 4 клетки или 8 пар.

## 3. Базовая геометрия

```text
Основной диск мира:
64 клетки = 16 дуг × 4 клетки.

Орбиталь:
16 клеток = 16 орбитальных позиций.

Одна клетка орбитали может проецироваться на одну дугу основного диска из 4 клеток.
```

Пример:

```text
Орбитальная клетка 1 накрывает основные клетки 1–4.
Орбитальная клетка 2 накрывает основные клетки 5–8.
...
Орбитальная клетка 16 накрывает основные клетки 61–64.
```

Так появляется не только точечное совпадение, но и зона влияния орбитали.

## 4. Два уровня совпадения

### 4.1. Зональное совпадение

Орбитальная клетка накрывает дугу из 4 клеток основного диска.

```text
Игрок стоит внутри дуги орбитального влияния → получает фоновый эффект орбитали.
```

### 4.2. Точное совпадение

Внутри дуги одна из 4 клеток считается точкой точной конъюнкции.

```text
Игрок стоит на точной клетке → запускается сильное событие / химическая реакция.
```

Это позволяет иметь постоянное влияние орбиталей без перегруза сильными событиями каждый ход.

## 5. Предлагаемая структура 64 клеток основного диска — DRAFT

64 клетки лучше не называть все вручную сразу. Рабочая структура:

```text
8 секторов × 8 клеток = 64 клетки.
```

Каждый сектор задаёт область мира. Каждая из 8 клеток внутри сектора задаёт функцию действия.

### 5.1. Восемь функций клетки

```text
1. Вход / появление
2. Ресурс / получение
3. Связь / контакт
4. Испытание / риск
5. Выбор / развилка
6. Преобразование / изменение
7. Порог / переход
8. Последствие / запись в кампанию
```

### 5.2. Восемь секторов физического диска — DRAFT

```text
1. Тело
2. Дом / территория
3. Дорога
4. Ресурс
5. Предмет / инструмент
6. Работа / создание
7. Конфликт
8. Порог / переход
```

Формула клетки:

```text
Сектор + функция = конкретная клетка.
```

Пример:

```text
Сектор: Ресурс.
Функция: Испытание.
Клетка: Спорный ресурс.
```

### 5.3. Восемь секторов тонкого диска — DRAFT

```text
1. Образ
2. Желание
3. Страх
4. Память
5. Сон
6. Связь
7. Искажение
8. Тонкая форма
```

### 5.4. Восемь секторов метагалактического диска — DRAFT

```text
1. Архетип
2. Имя
3. Закон
4. Луч
5. Иерархия
6. Сефир / узел древа
7. Миссия
8. Полномочие
```

## 6. Структура 16 клеток орбитали — DRAFT

Каждая орбиталь имеет 16 клеток, сгруппированных в 4 фазы по 4 клетки:

```text
Фаза 1: Вход / зов / первое влияние.
Фаза 2: Дар / ресурс / усиление.
Фаза 3: Испытание / цена / искажение.
Фаза 4: Переход / последствие / закрепление.
```

Шаблон 16 клеток любой орбитали:

```text
1. Зов орбитали
2. Знак
3. Контакт
4. Первое касание
5. Дар
6. Усиление
7. Союз / помощь
8. Открытие
9. Цена
10. Проверка
11. Искажение / ловушка
12. Долг / обязательство
13. Порог
14. Поворот орбитали
15. Закрепление
16. След в кампании
```

Для каждой конкретной орбитали эти 16 позиций получают собственную терминологию.

## 7. Механика движения

### 7.1. Основной диск

Игрок перемещается по 64-клеточному основному диску.

Варианты движения:

```text
Базовое движение: 1–4 клетки.
Быстрый ход: 5–8 клеток, но с риском.
Портальный ход: переход на связанную клетку другого диска.
Синтезный ход: действие сверху вниз без обычного перемещения.
```

### 7.2. Орбитали

Орбитали вращаются по 16 позициям.

Рабочий принцип:

```text
Каждая орбиталь имеет скорость вращения.
Скорость лучше делать разной: +1, +2, +3, +5 и т.д.
```

Важно: не все орбитали должны делить общий ритм. Иначе совпадения станут слишком предсказуемыми.

## 8. Карты и 64/16

### 8.1. Карты клеток мира

На каждую клетку можно не печатать отдельную карту. Дешевле и гибче:

```text
карта сектора + карта функции = конкретная клетка.
```

Пример:

```text
Сектор: Дорога.
Функция: Порог.
Результат: Перекрёсток перехода.
```

### 8.2. Карты орбитали

Для каждой орбитали нужен набор из 16 карт / эффектов, совпадающих с 16 позициями орбитального круга.

### 8.3. Карты событий

Событие рождается не одной картой, а формулой:

```text
Клетка основного диска + клетка орбитали + действие + инструмент + ресурс = событие.
```

## 9. Кубики в сетке 64/16

Кубики не должны случайно двигать всю систему вместо игрока. Их роль:

```text
1. качество действия;
2. цена действия;
3. риск искажения;
4. редкий импульс орбитали;
5. сила конъюнкции.
```

Рекомендуемые кубики:

```text
D6 действия
D6 риска
D8 силы события
D16 орбитальной позиции / редкого сдвига
```

D16 особенно подходит к орбиталям на 16 клеток.

## 10. Конъюнкции в сетке 64/16

### 10.1. Фоновое влияние

```text
Орбитальная клетка накрывает дугу из 4 клеток основного диска.
Игрок внутри этой дуги получает малый эффект.
```

### 10.2. Точная конъюнкция

```text
Если игрок стоит на точной клетке внутри дуги, событие усиливается.
```

### 10.3. Двойная конъюнкция

```text
Если две орбитали накрыли одну и ту же дугу, возникает химия двух орбиталей.
```

### 10.4. Великая конъюнкция

```text
Если основной диск, две или три орбитали и портал совпали на одной дуге, открывается переход или меняется закон зоны.
```

## 11. Практическое последствие для прототипа

Для первого бумажного теста не обязательно печатать 64 уникальные клетки. Можно сделать:

```text
1 основной диск на 64 позиции;
8 секторных карт;
8 функциональных карт;
3 орбитали по 16 позиций;
по 16 эффектов на каждую активную орбиталь;
4 персонажа;
4 кубика;
жетоны ресурсов / долгов / искажений / порталов / законов.
```
