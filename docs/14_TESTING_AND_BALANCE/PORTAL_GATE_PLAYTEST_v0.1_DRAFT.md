## 1. Цель

Проверить один узел до дальнейшего расширения игры:

> Создают ли кольца, спицы и конъюнкции содержательное решение о переходе без мёртвого времени, обязательного налога Накалом и управления всей группой одним игроком.

Это изолированный тест пропускной способности портала. Он не проверяет кампанию, лор, экономику Убежища, полные колоды ролей или баланс всех миров.

## 2. Комплект бумажного прототипа

- кольцо на 24 клетки;
- 6 равномерно расположенных спиц;
- центр диска;
- 2 вращающихся кольца;
- рабочие периоды 2 и 3;
- одно фиксированное Окно Совмещения;
- 4 фигурки игроков;
- по 3 жетона Накала каждому;
- 6 карт состояний конъюнкции;
- 8 карт угрозы/Искажения;
- трек из 8 тестовых раундов;
- лист наблюдений.

Начальные позиции фигурок назначаются так, чтобы каждая находилась в 2–5 шагах от ближайшей спицы. Это исключает проверку длинного маршевого движения: тестируется именно шлюз.

## 3. Общие правила

- каждый игрок выполняет одно движение и одно действие за раунд;
- кольца изменяют положение после фазы игроков;
- цель каждого игрока — войти в центр и выполнить один вертикальный переход;
- переход должен быть личным решением игрока;
- публично разрешено обсуждать общую угрозу, но тестер отмечает прямые указания другим игрокам;
- после перехода игрок продолжает выполнять упрощённое действие поддержки, чтобы тест не создавал искусственное исключение участника.

## 4. Сравниваемые версии

Каждая версия проводится минимум 3 раза. Порядок версий меняется между группами.

### Версия A — утверждённая модель качества перехода

- из центра перейти можно всегда;
- благоприятная конъюнкция: бесплатно и с бонусом;
- частичная: цена 1 Накал либо сброс карты;
- неблагоприятная: цена 2 Накала и карта угрозы;
- игрок может добровольно остаться в центре ради следующего окна, но обязан выполнить продуктивное действие.

### Версия B — старая модель зала ожидания

- перейти можно только при нужной конъюнкции;
- разрешено подкручивать кольцо Накалом;
- без нужной конъюнкции игрок остаётся в центре.

Эта версия нужна только как контроль для подтверждения причины отказа. Она не возвращается в основной дизайн без нового решения основателя.

### Версия C — спицы без временного запрета

- войти в центр можно только через спицу;
- переход из центра мгновенный и не зависит от колец;
- конъюнкции дают только бонусы и угрозы зоны.

Версия C показывает, создаёт ли сама система качества перехода дополнительную ценность по сравнению с одной навигацией.

## 5. Метрики одной партии

Фиксировать для каждого игрока:

- раунд принятия решения «иду на переход»;
- раунд фактического перехода;
- число ходов в центре;
- число вынужденных и добровольных ожиданий;
- Накал, потраченный на переход;
- Накал, потраченный только для устранения ожидания;
- выбранное качество перехода;
- полученные последствия;
- кто предложил решение;
- изменил ли игрок своё решение после совета другого участника.

Фиксировать для группы:

- сколько игроков перешли до конца 8-го раунда;
- сколько времени заняло объяснение механики;
- сколько раз кольца реально изменили решение;
- сколько раз игроки заранее вычислили расписание без нового выбора;
- число прямых команд одного игрока другим;
- субъективную оценку напряжения и раздражения по шкале 1–5;
- какую версию группа хочет сыграть повторно.

## 6. Критерии провала

Механика требует переделки, если выполняется хотя бы одно условие:

- медиана вынужденного ожидания превышает 0 раундов в версии A;
- хотя бы один игрок теряет более одного раунда только из-за состояния колец;
- более 15% действий не содержат нового выбора;
- более 25% всего Накала тратится только для устранения простоя;
- менее 75% игроков завершают переход до конца теста;
- один участник определяет маршруты и тайминг более чем в 30% чужих ходов;
- игроки предпочитают версию C версии A, потому что кольца воспринимаются как административная надстройка;
- объяснение качества переходов занимает более 5 минут сверх базового движения.

## 7. Критерии предварительного успеха

Механика допускается к следующей итерации, если после минимум 9 партий:

- версия A имеет медиану 0 вынужденных ожиданий;
- минимум 80% переходов включают реальный выбор между ценой, риском и подготовкой;
- Накал используется минимум в двух разных функциях, а не только для подкрутки;
- версия A получает более высокий запрос повторной партии, чем версия B;
- версия A отличается от версии C содержательными решениями, а не только дополнительным подсчётом;
- индекс альфа-давления соответствует TEST-001: не менее 70% ходов решает сам активный игрок.

## 8. Отдельные проверки ролей

Роли вводятся только после первых трёх безролевых партий.

### Провидец

Проверить два варианта:

- видит одно будущее состояние;
- видит два будущих состояния.

Провал: роль либо не влияет на решения, либо начинает диктовать оптимальные ходы всей группе.

### Проводник Портала

Проверить действия:

- удержать окно;
- провести второго игрока;
- перенаправить выход;
- снизить опасность на одну ступень.

Провал: без Проводника группа системно не может завершить переходы.

### Страж

Проверить ограничение силы от груза:

- потолок бонуса;
- штраф к движению;
- повышение угрозы.

Провал: накопленный груз одновременно увеличивает силу и облегчает дальнейшее накопление без встречной цены.

## 9. Дисциплина теста

- не менять числа внутри серии из трёх партий;
- не объяснять игрокам, какая версия считается предпочтительной;
- записывать дословные жалобы на ожидание, расчёты и непонятные рецепты;
- не добавлять новые миры или эффекты ради «спасения» слабой версии;
- после каждой серии составлять таблицу «было → результат → решение»;
- отрицательный результат не переигрывать до получения желаемого исхода.
