## 0. Правила исполнения любого промпта

**0.1. Обязательный контекст (DEC-008).** Перед исполнением любого промпта читаются: `PROJECT_CORE_CONTEXT.md`, `DECISIONS.md`, релевантный раздел `docs/`, `MIGRATION_INDEX.md`. Проверяются версии. Результат сохраняется в целевой файл, запись — в `CHANGELOG.md`.

**0.2. Статусная дисциплина.** Любой результат промпта — DRAFT / HYPOTHESIS. Новая гипотеза не заменяет утверждённое решение (DEC-001…008). Если результат требует изменения ядра — оформляется предложение в «Запись в журнал решений», решение принимает основатель.

**0.3. Терминологический файрвол.** Во всех игровых результатах используется только внутренний словарь IP: Диск Мира, Низведение, Извержение, Эхо, Тень, Луч, Якорь, Импульс, Ритуал, Исток, Высь / Спектр / Твердь / Глубь, Искажение, Пожиратель, Ядро, Прокол / Форсаж / Разложение / Магнит. Термины первоисточников допустимы только в Блоках 1–2 и в `references/` (внутренний контур).

**0.4. Чёрный список механик.** Ни один результат не может содержать: Feedback Snowball (усиление лидера), Status Claiming / права вето и иерархические привилегии над игроками, 16-ярусные системы сложности, тёмные паттерны таймеров вовлечения. Промпт, требующий их, не исполняется — фиксируется отказ и причина.

**0.5. Фильтр DEC-006.** Каждая предлагаемая механика проверяется по пяти осям: почему выбрана, эмоциональный эффект, взаимодействие с другими механиками, риски, защита баланса. Приоритет №1 — отсутствие эффекта убегающего лидера.

**0.6. Стандартный формат ответа — 12 секций** (как в `WORLD_ARCHITECTURE_v0.1_DRAFT.md`):
1. Резюме · 2. Источники и версии · 3. Каноническое ядро · 4. Сравнения · 5. Варианты · 6. Рекомендуемая архитектура · 7. Спецификация · 8. Пример · 9. Риски · 10. Прототип и тестирование · 11. TO-VERIFY · 12. Запись в журнал решений.

**0.7. Универсальный контекст-заголовок** (вставляется перед текстом любого промпта):

> Ты — [роль-владелец блока] студии IVDIVO GAME MASTER. Прочитай PROJECT_CORE_CONTEXT.md, DECISIONS.md и целевой раздел docs/. Работай строго внутри утверждённого концепта (концепт-фриз): ничего не добавляй в ядро, организуй и углубляй существующее. Ответ — в 12-секционном формате. Статус результата — DRAFT. Соблюдай правила 0.1–0.6.
