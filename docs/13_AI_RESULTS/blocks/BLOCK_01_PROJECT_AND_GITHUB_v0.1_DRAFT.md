Разрешённая коллизия:

- ранний DEC-003 и WORLD-001 описывали восемь вертикальных миров;
- DEC-015 и WORLD-002 утвердили пять вертикальных миров и орбитали параллельных миров;
- DEC-003 теперь SUPERSEDED;
- WORLD-001 сохраняется как исторический DRAFT;

Другие риски репозитория:

- несколько GitHub PR образуют stacked-цепочку и не должны сливаться в случайном порядке;
- Google Drive содержит архивные дубликаты протоколов и журналов решений;
- продуктовая последовательность дополнений остаётся REVIEW и не должна смешиваться с утверждённым каноном.

## 4. Источник истины

Порядок:

1. прямое решение основателя;
2. более поздний FOUNDER-DECISION;
3. APPROVED;
4. REVIEW;
5. DRAFT;
6. AI PROPOSAL.

Для архитектуры миров действуют DEC-015 и WORLD-002.

Числа не получают APPROVED без теста. Канонические имена не создаются ИИ без источника.

## 5. Зависимости

Критический граф:

`WORLD-002 → BASE SCOPE → CORE LOOP → ROLE/CARD PROTOTYPE → PLAYTEST → CAMPAIGN → EXPANSIONS → ART/PRODUCTION → KICKSTARTER`.

Нельзя проектировать 50 финальных иллюстраций до блокировки функций карт. Нельзя балансировать дополнение до работающей базы.

## 6. Журнал решений

Зафиксировано DEC-015:

> Пять вертикальных миров образуют ось восхождения; параллельные миры располагаются на орбиталях и не являются ступенями вертикали.

Отдельное продуктовое направление остаётся REVIEW:

> Базовая коробка является законченной игрой; полные вертикальные миры и орбитали вводятся модульными дополнениями после освоения ядра.

## 7. Гейты

- Gate A: канон не содержит активных противоречий — **выполнено для архитектуры миров**.
- Gate A2: составлена карта привязки орбиталей — OPEN.
- Gate B: два игрока проходят обучающий сценарий без помощи автора.
- Gate C: core loop даёт содержательное решение каждые 3–5 минут.
- Gate D: портал создаёт выбор без вынужденного ожидания.
- Gate E: первое открытие меняет старую карту и используется в следующем сценарии.
- Gate F: минимум 40 слепых партий перед Kickstarter-разработкой финального масштаба.

## 8. Терминология

Создать трёхслойный словарь:

- внутренний источник;
- дизайнерский термин;
- player-facing термин.

Каноническая основа словаря миров: Физический, Тонкий, Метагалактический, Огненный, Синтезный; орбитальный класс: Ад, Рай, Аидия, Демонский, Омарный и другие TO-VERIFY.

Запрещено переносить эзотерическую терминологию напрямую в массовый продукт без решения.

## 9. Формы

Обязательны шаблоны: механика, мир, дополнение, персонаж, фракция, карта, сценарий, тест, art brief, production spec.

Паспорт орбитального мира обязан включать связь с вертикалью, экономику, жителей, фракции, тип конъюнкции и цену перехода.
