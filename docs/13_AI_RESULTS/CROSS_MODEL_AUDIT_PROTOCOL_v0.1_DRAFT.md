## Назначение

Замена методу "мегапромпт с 30 критериями" для задачи именно поиска пробелов (не для маркетингового сравнения с рынком — это отдельная легитимная задача, см. COMP-004). Разработан после того, как выяснилось: промпты с 30 оценочными критериями провоцируют "иллюзорную триангуляцию" — если ни GPT, ни Claude не видели исходный документ, обе модели могут независимо изобрести похожую правдоподобную механику для одного и того же названия на основе жанровой статистики, и совпадение ничего не подтверждает.

## Три поломки утвердительной схемы

1. Утвердительная рамка вместо проверочной ("оцени экономику Х по 10 параметрам" уже предполагает, что Х существует).
2. Иллюзорная триангуляция (см. выше).
3. Смешение двух разных задач — бенчмаркинг качества и инвентаризация существования.

## Протокол: три фазы

**Фаза 0 — Паритет исходников.** Один и тот же необработанный текст документа (не пересказ) вставляется в обе модели. Ни одна не работает по памяти о прошлых своих резюме.

**Фаза 1 — Инвентаризация без оценки.** Обеим моделям независимо даётся один и тот же список терминов для проверки. Задача — заполнить таблицу: Термин | Найден в тексте? (да/нет) | Дословная цитата или "НЕ НАЙДЕНО" | Есть числовое/механическое правило, или только название. Жёсткое правило: без дословной цитаты статус обязан быть "НЕ НАЙДЕНО", даже при уверенности модели, что видела нечто похожее.

**Фаза 2 — Независимый список пробелов "снизу вверх".** Каждая модель, глядя только на свою таблицу Фазы 1, составляет список пробелов из статусов "не найдено"/"только название" — не из внешнего чек-листа, придуманного заранее.

**Фаза 3 — Сверка.** Три категории: (а) пробел отмечен обеими моделями — высокая уверенность, независимая сходимость от разных прочтений одного текста; (б) отмечен только одной — требует ручной проверки (попросить привести цитату повторно); (в) не отмечен ни одной, но известен человеку — самая ценная категория: информация существует только в голове автора, ни одна кросс-модельная проверка это не найдёт, единственный способ закрыть — записать вручную.

## Практическое отличие

Старая схема: один промпт → гладкий, местами придуманный отчёт на 30 пунктов. Новая: скучная таблица цитат → список реальных дыр → сверка, где несовпадение — сигнал проверить руками, совпадение — единственное, чему стоит доверять в подобной кросс-модельной проверке.

## Что это не отменяет

Маркетинговое сравнение с рынком (COMP-004) остаётся легитимной отдельной задачей, но должно запускаться ПОСЛЕ этого протокола и только по терминам, которые Фаза 3 подтвердила как реально существующие.
